# UI Wireframes & Screen Layouts

## Overview

This document provides wireframe layouts for all major screens in Primordial Swamp. These wireframes serve as reference for UI implementation and agentic development.

---

## Screen Dimensions

- **Target Resolution:** 1280 x 720 (16:9)
- **Safe Area:** 40px padding from edges
- **Touch Target Minimum:** 44x44 pixels

## Responsive Considerations

### UI Scaling
The game supports UI scaling from 100% to 200%:

| Scale | Use Case |
|-------|----------|
| 100% | Standard desktop play |
| 125% | Large monitors, accessibility |
| 150% | Small screens, visibility needs |
| 200% | Low vision, touch devices |

### Touch Targets
All interactive elements maintain minimum 44x44px touch targets:
- Buttons: Padding ensures target size
- Map nodes: 50px diameter
- Sliders: Full track height interactive

---

## Color Usage Guidelines

### Primary Actions
- **Green (#4a9d5f):** Start, Confirm, Positive
- **Hover:** White text on green background

### Secondary Actions  
- **Gray (#888888):** Back, Cancel, Secondary
- **Hover:** White text on gray background

### Semantic Colors
- **Red (#d94a3d):** Damage, Danger, Enemies
- **Blue (#4a8bd9):** Stamina, Rest, Info
- **Yellow (#e8a735):** Warning, Events
- **Purple (#9d4a9d):** Bosses, Special

---

## Animation Guidelines

### Transitions
- **Scene transitions:** 500ms fade
- **Button hover:** 200ms ease
- **Panel appear:** 300ms slide

### Feedback
- **Hit effect:** 200ms screen shake
- **Damage number:** Float up, fade out
- **Level up:** Scale pulse

### Reduced Motion Mode
When enabled:
- No screen shake
- Instant transitions
- Static feedback indicators
